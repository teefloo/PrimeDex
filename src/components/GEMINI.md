# 📔 PrimeDex: Component Architecture & Guidelines

This directory (`src/components`) contains the modular UI building blocks of the PrimeDex application. All components must adhere to the high-performance and visual-fidelity standards defined in the project's root mandates.

## 📁 Directory Structure

### 1. `layout/`
Contains global and structural components that define the application's shell.
- **Key Components**: `Header.tsx` (sticky, glassmorphism), `HeroSection.tsx`, `SettingsModal.tsx`.
- **Convention**: These components typically manage global state transitions (theme, language, settings visibility).

### 2. `pokemon/`
Domain-specific components for Pokemon data display and interaction.
- **Key Components**: 
  - `PokemonList.tsx`: Handles infinite scrolling and advanced filtering logic via TanStack Query.
  - `PokemonCard.tsx`: The primary data entry, featuring animations, hover effects, and action buttons.
  - `AdvancedFilters.tsx`: Complex filter panel with multiple state triggers.
- **Convention**: Use specialized hooks from `src/store/primedex` for atomic state selectors to minimize re-renders.

### 3. `ui/`
Reusable primitive components built with `@base-ui/react` and `class-variance-authority` (CVA).
- **Style**: Tailwind CSS 4 + custom glassmorphism utilities.
- **Key Components**: `button.tsx`, `card.tsx`, `dialog.tsx`, `skeleton.tsx`.

## 🛠 Tech Stack & Patterns

- **Framework**: Next.js 16 (App Router) + React 19.
- **State Management**: Zustand (`usePrimeDexStore`) for persistence and UI state.
- **Data Fetching**: TanStack Query v5 with aggressive caching.
- **Animations**: Framer Motion for transitions and hover states.
- **Localization**: `react-i18next` for multi-language support. All UI strings must use the `t()` function.

## 📏 Engineering Standards

1.  **Performance**:
    - Use `'use client'` only when necessary for interactivity.
    - Memoize heavy components like `PokemonCard` to prevent unnecessary re-renders in large lists.
    - Prefer `next/image` for all artwork to ensure optimized loading and blur placeholders.

2.  **Visual Polish**:
    - Adhere to the "Dark-Mode-First" aesthetic.
    - Use the `glass-panel`, `holo-border`, and `type-glow` CSS classes for consistent styling.
    - All interactive elements should have hover/active states defined via Framer Motion.

3.  **Accessibility (A11y)**:
    - Every interactive element (buttons, links) MUST have an `aria-label`.
    - Use the `Tooltip` component for icon-only buttons.
    - Ensure keyboard navigability (focus rings are mandatory).

4.  **SEO**:
    - Implement JSON-LD structured data (Schema.org) within list and detail components for rich search results.

## 🚀 Development Workflow

- **Styles**: Use Tailwind 4 theme variables (e.g., `text-primary`, `bg-secondary/10`).
- **Icons**: Use `lucide-react` for consistent iconography.
- **Validation**: Ensure type rigor; avoid `any` at all costs. Reference `src/types/pokemon.ts`.
